# 333 项目开发结构目录说明

## 1. 总体架构

- 前端：Vue3 + Pinia + Vue Router + Element Plus
- 后端：FastAPI + SQLAlchemy + JWT + 向量检索
- 聊天模式：
  - `free`：基于模型自身能力直接回答
  - `rag_selected`：先检索用户选中文档，再基于检索结果回答

---

## 2. 后端结构（backend）

### 2.1 应用入口与路由装配
- 文件：`main.py`
- 结构：应用装配层（Application Layer）
- 实现方式：
  - 创建 FastAPI 应用
  - 配置 CORS
  - 挂载 `user/documents/chat` 路由到 `/api/*`
  - 启动时自动建表

### 2.2 配置管理
- 文件：`config.py`
- 结构：集中配置对象（Settings Object）
- 实现方式：
  - 通过 `BaseSettings` 读取 `.env`
  - 统一管理 JWT、数据库、上传、模型、向量库参数

### 2.3 数据库基础设施
- 文件：`database.py`
- 结构：基础设施层（Infrastructure）
- 实现方式：
  - 初始化 SQLAlchemy `engine` 与 `SessionLocal`
  - 通过 `get_db()` 依赖注入 DB 会话

### 2.4 数据模型
- 文件：`models.py`
- 结构：ORM 领域实体（Domain Entities）
- 实现方式：
  - 用户：`User`
  - 文档：`Document`
  - 会话：`ChatSession`
  - 消息：`ChatMessage`
  - 用户 AI 配置：`UserAIConfig`
  - 通过 relationship 建立一对多关联

### 2.5 认证与用户
- 文件：`auth.py`、`routers/user.py`
- 结构：认证服务 + 用户路由
- 实现方式：
  - `auth.py` 负责密码哈希、JWT 生成/校验、当前用户解析
  - `user.py` 提供登录、注册、信息、改密、刷新 token、AI 配置接口

### 2.6 文档管理与处理
- 文件：`routers/documents.py`、`services/document_processor.py`
- 结构：路由编排 + 后台任务 + 文本处理服务
- 实现方式：
  - 上传后立即返回，后台异步处理文档（FastAPI BackgroundTasks）
  - 文本处理链路：提取文本 -> 分块 -> 写入向量库
  - 文档状态流转：`processing` / `completed` / `failed`
  - 支持列表、详情、状态、删除、重处理

### 2.7 向量检索抽象
- 文件：`services/vector_store.py`（以及 `faiss/chroma/simple/mock` 实现）
- 结构：策略模式（Strategy）+ 回退（Fallback）
- 实现方式：
  - 根据配置动态加载向量后端
  - 若主后端初始化失败，自动回退到 fallback

### 2.8 对话系统（核心）
- 文件：`routers/chat.py`、`services/ai_service.py`
- 结构：路由分流 + 服务层生成 + SSE 流式输出
- 实现方式：
  - `chat.py` 根据 `chat_mode` 分流：`free` 或 `rag_selected`
  - `rag_selected` 下先按选中文档检索向量片段
  - `ai_service.py` 按模式选择不同 system prompt：
    - `FREE_SYSTEM_PROMPT`
    - `RAG_SYSTEM_PROMPT + 参考文档上下文`
  - 支持流式 `/chat/ask` 与同步 `/chat/ask/sync`
  - 会话与消息入库，记录 sources 与 response_time

## 3. 前端结构（frontend/src）

### 3.1 应用启动
- 文件：`main.js`
- 结构：前端启动装配层
- 实现方式：
  - 挂载 Pinia、Router、Element Plus

### 3.2 路由系统
- 文件：`router/index.js`
- 结构：声明式路由 + 路由守卫
- 实现方式：
  - 路由页：Login / Dashboard / Documents / Chat / Settings
  - 通过 `meta.requiresAuth` 与守卫进行鉴权跳转

### 3.3 全局状态
- 文件：`store/index.js`
- 结构：Pinia Store（用户域 + 聊天域）
- 实现方式：
  - 用户域：登录态、token、游客模式、用户信息
  - 聊天域：当前会话、聊天记录、流式状态

### 3.4 API 访问层
- 文件：`api/auth.js`、`api/chat.js`、`api/documents.js`
- 结构：按业务拆分的 API Client
- 实现方式：
  - `auth.js`：axios 实例 + 拦截器 + 401 刷新 token
  - `chat.js`：`fetch` + SSE 逐 token 解析
  - `documents.js`：文档上传/查询/状态/重处理/删除

### 3.5 页面实现
- 文件：`views/*.vue`
- 结构：视图容器组件（View-driven）
- 实现方式：
  - `Chat.vue`：会话列表、消息流、模式切换、文档选择、流式渲染
  - `Documents.vue`：文档管理与状态跟踪
  - `Settings.vue`：模型/API 配置
  - `Login.vue`：登录注册/游客入口
  - `Dashboard.vue`：总览信息

## 4. 双模式对话的实现要点（当前版本）

1. 模式入口统一在请求参数 `chat_mode`。
2. 前端发送：
   - `free`：只传 `chat_mode=free`
   - `rag_selected`：同时传 `selected_document_ids`
3. 后端校验：
   - `rag_selected` 必须登录且必须选择文档
4. 服务层生成：
   - `free` 使用通用助手提示词
   - `rag_selected` 使用知识库提示词 + 检索上下文
5. 响应层：
   - 流式 SSE 输出 token
   - 完成后回传 `session_id` 和 `sources`
